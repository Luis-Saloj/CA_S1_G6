# Acuerdos y Trade-offs

## Introducción
Durante la simulación de la junta directiva técnica, el equipo discutió distintas soluciones para los problemas identificados en la Farmacia del Ángel.  

Como resultado de esta discusión, se alcanzaron acuerdos clave, tomando en cuenta las limitaciones del cliente, especialmente en términos de presupuesto, tiempo y necesidades del negocio.

---

## Acuerdos alcanzados

### Acuerdo 1: Implementación de un sistema modular
Se decidió desarrollar un sistema modular en lugar de un sistema completo desde el inicio.  

Esto permite:
- Implementar funcionalidades por fases  
- Reducir el costo inicial  
- Adaptarse al crecimiento del negocio  

---

### Acuerdo 2: Uso de modelo tipo servicio (mensualidad)
Se acordó ofrecer el sistema bajo un modelo de pago mensual, en lugar de un pago único.  

Esto permite:
- Facilitar el acceso al sistema  
- Adaptarse al presupuesto limitado del cliente  
- Generar ingresos continuos  

---

### Acuerdo 3: Priorizar problemas críticos
Se decidió enfocarse primero en los problemas más importantes:
- Digitalización de ventas  
- Control de inventario  
- Apoyo en conocimiento de medicamentos  

Esto permite generar valor desde las primeras etapas del sistema.

---

### Acuerdo 4: Entregas parciales del sistema
Se acordó mostrar avances periódicos al cliente.  

Esto permite:
- Validar que el sistema cumpla con sus necesidades  
- Realizar ajustes durante el desarrollo  
- Mantener comunicación constante  

---

## Trade-offs identificados

### Trade-off 1: Tiempo vs Calidad
- Se solicitó una entrega rápida del sistema  
- Sin embargo, el equipo técnico indicó que se requiere más tiempo para evitar errores  

**Decisión:**  
Se extendió el tiempo de desarrollo para asegurar mejor calidad del sistema.

---

### Trade-off 2: Costo vs Funcionalidad
- El cliente cuenta con un presupuesto limitado  
- Un sistema completo tendría un costo elevado  

**Decisión:**  
Se optó por un sistema modular con funcionalidades básicas iniciales.

---

### Trade-off 3: Simplicidad vs Complejidad técnica
- Se propuso un sistema completo tipo SaaS  
- Esto implicaba mayor complejidad y tiempo de desarrollo  

**Decisión:**  
Se eligió una solución más simple y progresiva, iniciando con módulos básicos.

---

### Trade-off 4: Inteligencia Artificial vs Confiabilidad
- Se propuso el uso de inteligencia artificial para recomendaciones  
- Sin embargo, existe el riesgo de errores en la información  

**Decisión:**  
La inteligencia artificial se utilizará solo como apoyo informativo, no como fuente principal de decisión.

---

### Trade-off 5: Sistema en la nube vs sistema local
- El sistema en la nube permite menor costo y mayor escalabilidad  
- El sistema local implica mayor costo pero mayor independencia  

**Decisión:**  
Se optó por un sistema en la nube para reducir costos, aceptando la dependencia de conexión a internet.

---

## Conclusión
Los acuerdos alcanzados reflejan un equilibrio entre las necesidades del cliente y las capacidades del equipo técnico.  

Los trade-offs permitieron tomar decisiones realistas, priorizando:
- accesibilidad económica  
- funcionalidad esencial  
- calidad del sistema  
- viabilidad del desarrollo  

Esto demuestra un proceso de negociación técnica adecuado y alineado con un entorno real de desarrollo de software.
