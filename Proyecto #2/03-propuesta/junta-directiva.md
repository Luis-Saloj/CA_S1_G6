# Simulación de Junta Directiva Técnica

## Introducción
En esta sección se presenta la simulación de una junta directiva técnica, basada en los problemas identificados en la Farmacia del Ángel.  

El objetivo de esta simulación fue analizar distintas propuestas de solución, discutirlas desde diferentes perspectivas y llegar a un acuerdo viable considerando tiempo, costo y calidad.

---

## Contexto del problema
A partir de la entrevista realizada, se identificaron múltiples problemas en la farmacia, entre los más importantes:

- Uso de registros manuales  
- Falta de conocimiento sobre medicamentos  
- Dificultad en dosificación para niños  
- Falta de registro de clientes  
- Problemas en inventario  
- Falta de tiempo para mejoras tecnológicas  

Estos problemas fueron el punto de partida para la discusión técnica.

## Desarrollo de la discusión

### 1. Planteamiento inicial
Se presentaron los principales problemas del negocio, destacando:

- Registros manuales de ventas  
- Falta de digitalización  
- Problemas en inventario  
- Falta de conocimiento sobre medicamentos  

El equipo coincidió en que estos problemas requerían una solución tecnológica.

---

### 2. Conflicto principal

Durante la discusión surgieron varios conflictos importantes:

#### Tiempo vs Calidad
- El Product Manager solicitaba una solución rápida  
- El Tech Lead y QA indicaban que se necesitaba más tiempo para evitar errores  

#### Presupuesto del cliente
- El cliente indicó que no cuenta con suficiente presupuesto  
- Se discutió si desarrollar un sistema completo o una solución más accesible  

#### Seguridad de la información
- El cliente expresó preocupación sobre compartir datos  
- Se debatió sobre el uso de un sistema compartido (tipo SaaS)  

#### Uso de inteligencia artificial
- Se propuso usar IA para recomendaciones de medicamentos  
- Se cuestionó su precisión y confiabilidad  

---

### 3. Propuestas de solución

Se plantearon varias soluciones:

- Sistema completo tradicional (rechazado por costo)  
- Sistema tipo SaaS (servicio por mensualidad)  
- Sistema modular (aceptado como mejor opción)  

**Propuesta final:**

Crear un sistema modular que permita:
- Implementar funciones poco a poco  
- Reducir costos iniciales  
- Adaptarse al crecimiento del negocio  

---

### 4. Solución técnica propuesta

El desarrollador propuso una solución basada en:

- Sistema web centralizado  
- Base de datos separada por cliente  
- Módulos independientes (inventario, ventas, clientes, etc.)  
- Plantilla base ya desarrollada  

También se propuso:

- Uso de IA para apoyo informativo (no crítico)  
- Sistema simple para orientación de medicamentos  
- Control de inventario automatizado  

---

### 5. Resolución del conflicto

Se llegó a los siguientes acuerdos:

- Utilizar un sistema modular en lugar de uno completo  
- Implementar primero los problemas más críticos  
- Mantener costos bajos mediante modelo de mensualidad  
- No depender totalmente de IA, sino usarla como apoyo  
- Priorizar facilidad de uso para el cliente  

---

### 6. Plan de trabajo acordado

Se definió el siguiente plan:

- Entrega aproximada en 1 mes  
- Entregas parciales (avances semanales)  
- Fase de pruebas adicional antes de entrega final  
- Posibilidad de agregar mejoras futuras  

---

## Conclusión

La simulación permitió evidenciar la importancia de la comunicación asertiva y la negociación técnica en el desarrollo de soluciones.  

Se logró llegar a una solución viable que equilibra:

- necesidades del cliente  
- limitaciones económicas  
- calidad del sistema  
- tiempo de desarrollo  

Esto demuestra la capacidad del equipo para trabajar en conjunto y tomar decisiones técnicas en un entorno realista.
