# Requisitos del Sistema

## Introducción

Los siguientes requisitos fueron identificados a partir de la entrevista realizada en la Farmacia del Ángel, con el objetivo de proponer una solución tecnológica que permita mejorar la eficiencia del negocio, reducir errores y optimizar la atención al cliente.

---

## Requisitos Funcionales

### Gestión de productos e inventario

- **RQ-01:** El sistema debe permitir registrar productos con nombre, descripción (con IA), categoría y precio.  
- **RQ-02:** El sistema debe almacenar la fecha de vencimiento de cada producto.  
- **RQ-03:** El sistema debe permitir visualizar el inventario disponible.  
- **RQ-04:** El sistema debe alertar cuando un producto esté próximo a vencer.  
- **RQ-05:** El sistema debe alertar cuando un producto tenga bajo stock.  

---

### Gestión de ventas

- **RQ-06:** El sistema debe permitir registrar ventas de productos con lector de código de barras.  
- **RQ-07:** El sistema debe almacenar un historial de ventas.  
- **RQ-08:** El sistema debe automatizar el registro de ventas, eliminando el uso de registros manuales.  

---

### Gestión de clientes

- **RQ-09:** El sistema debe permitir registrar información básica de clientes.  
- **RQ-10:** El sistema debe permitir consultar el historial de compras de un cliente.  

---

### Apoyo en medicamentos

- **RQ-11:** El sistema debe proporcionar información básica sobre los medicamentos (uso, función, recomendaciones).  
- **RQ-12:** El sistema debe permitir orientar al usuario sobre la dosificación de medicamentos para niños (edad, peso, etc.).  
- **RQ-13:** El sistema debe mostrar información simplificada para facilitar la explicación a los clientes.  

---

### Seguridad y control

- **RQ-14:** El sistema debe permitir la gestión de usuarios (administrador, empleado).  
- **RQ-15:** El sistema debe evitar errores en la venta de medicamentos mediante validaciones.  
- **RQ-16:** El sistema debe registrar quién realiza cada acción en el sistema.  

---

### Usabilidad y eficiencia

- **RQ-17:** El sistema debe ser fácil de usar para el encargado del negocio.  
- **RQ-18:** El sistema debe reducir el tiempo necesario para realizar tareas repetitivas.  
- **RQ-19:** El sistema debe permitir el acceso rápido a la información de productos.  

---

## Requisitos No Funcionales

- **RNF-01:** El sistema debe ser rápido y eficiente en el registro de información.  
- **RNF-02:** El sistema debe ser confiable y evitar pérdida de datos.  
- **RNF-03:** El sistema debe ser seguro para proteger la información del negocio.  
- **RNF-04:** El sistema debe tener una interfaz clara y comprensible.  
- **RNF-05:** El sistema debe ser accesible desde dispositivos comunes (computadora o navegador web).  

---

## Conclusión/Observaciones

Los requisitos definidos permiten estructurar una solución que responde directamente a los problemas identificados en la Farmacia del Ángel, asegurando una mejora en la gestión del negocio y en la atención al cliente. Esperando proporcionar un servicio menusal y no vender el sistema completo para ayudar a presupuesto para el cliente, esto con un Saas, así poder ayudar a otros negocios con requisitos parecido y poder moldear el sistema para cada negocio.
